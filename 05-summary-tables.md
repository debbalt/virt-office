> Сводные таблицы событий и эндпоинтов.

### IN: 1С → Site

Все события приходят через `POST /intake/` (§3) с единой оболочкой `{ event_type, payload }`. Дополнительных транспортных полей на уровне envelope нет: идемпотентность ведётся только по доменным ключам (см. §1.4).


| event_type                          | Описание                                                                                                                                                                                                                                                                         | Поле идемпотентности                      | Уведомление                                                                                                            |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `user.create`                       | Создать пользователя в обычном (не корпоративном) кабинете                                                                                                                                                                                                                       | `login`                                   | Magic-link email                                                                                                       |
| `user.update`                       | Патч записи пользователя / компании по `login`: контакты, `**name**`, `**organization_name**`, `**scan_rule**`. Поле `**changed_at**` обязательно — LWW против OUT `**POST /users/sync/**` (§4.2). Отдельных событий `corporate.company.update` / `company.scan_rule.update` нет | `login`                                   | При изменении `**scan_rule**` с 1С → внутренний `scan_rule.changed_by_manager` (если значение реально сменилось, §3.6) |
| `corporate.cabinet.create`          | Создать **только** кабинет с реквизитами мастера. Компании в кабинет добавляются отдельными `corporate.company.attach` (одно событие = одна операция). Поле `companies[]` запрещено (`400 companies_field_not_allowed`)                                                          | `cabinet_id`                              | Invite email мастеру                                                                                                   |
| `corporate.cabinet.update`          | Обновить **только реквизиты** кабинета и/или мастера; смена `master_email` = передача мастерства, бывший мастер **полностью отвязывается**. Поле `companies[]` запрещено (`400 companies_field_not_allowed`)                                                                     | `cabinet_id`                              | Invite email при смене мастера, если новый ещё не зарегистрирован                                                      |
| `corporate.user.invite`             | Пригласить сотрудника в кабинет; стартовая видимость задаётся в `company_logins[]` (опц., default — пустой). Один email может быть мастером в одних кабинетах и member в других                                                                                                  | `cabinet_id + email`                      | Invite email                                                                                                           |
| `corporate.user.access.update`      | Обновить видимость компаний у сотрудника (IN-копия §4.9; LWW по `synced_at`; master не редактируется → `409 master_access_immutable`)                                                                                                                                            | `cabinet_id + email` (snapshot)           | Пересчёт fan-out по §3.6 для этого сотрудника                                                                          |
| `corporate.company.attach`          | Привязать **уже существующий** на сайте `login` к корп. кабинету (`cabinet_id`); только членство, без `name` / `organization_name` / `scan_rule`. Если пользователя нет → `404 user_not_found`; если компания в другом кабинете → `409 company_in_other_cabinet` (transfer)      | `login`                                   | Нет (стартовая видимость = 0; видимость — §3.2 / §4.9)                                                                 |
| `corporate.company.transfer`        | Атомарный перевод компании (`login`) из `from_cabinet_id` в `to_cabinet_id`. Видимость сотрудникам целевого кабинета **не** задаётся (как при `attach`; отдельно §4.9 / `corporate.user.access.update`). Каскадное удаление из access только у исходного кабинета                | `(login, to_cabinet_id)`                  | Нет (видимость при необходимости — отдельным access-снимком)                                                           |
| `corporate.company.detach`          | Отвязка компании от кабинета в индивидуальный режим; сайт создаёт individual user и шлёт magic-link установки пароля на `owner_email`                                                                                                                                            | `login`                                   | Magic-link установки пароля; повтор без изменений — без письма                                                         |
| `contract.create`                   | Создать карточку договора (только данные, без файла скана)                                                                                                                                                                                                                       | `contract_id`                             | Нет                                                                                                                    |
| `contract.update`                   | Обновить данные договора (без файла скана)                                                                                                                                                                                                                                       | `contract_id`                             | Нет                                                                                                                    |
| `contract.scan.upload`              | Загрузить или заменить файл скана договора                                                                                                                                                                                                                                       | `contract_id + login`                     | Внутренний fan-out по §3.6 (`contract.scan_uploaded`)                                                                  |
| `storage_location.upsert`           | Объект аренды = адрес хранения (структурные данные: город, район, адрес, тип пакета); тексты/фото/фичи — в админке сайта                                                                                                                                                         | `location_id`                             | Нет                                                                                                                    |
| `package.upsert`                    | Стабильный код типа пакета + флаг `is_active`. Названия, описания, иконки, бейджи и порядок отображения — в админке сайта                                                                                                                                                        | `package_id`                              | Нет                                                                                                                    |
| `pricing.upsert`                    | Snapshot одного документа `Установка цен на услуги аренды`: (`customer_category=client` × `package_id` × диапазон месяцев) + цены по объектам                                                                                                                                    | `pricing_document_id`                     | Нет                                                                                                                    |
| `pricing.delete`                    | Удалить ранее загруженный документ цен                                                                                                                                                                                                                                           | `pricing_document_id`                     | Нет                                                                                                                    |
| `mail.create`                       | Новое письмо в кабинете                                                                                                                                                                                                                                                          | `message_id`                              | Внутренний fan-out по §3.6 (`mail.new`)                                                                                |
| `mail.update`                       | Обновить данные письма                                                                                                                                                                                                                                                           | `message_id`                              | Нет                                                                                                                    |
| `mail.status.update`                | Обновить статус письма                                                                                                                                                                                                                                                           | `message_id + status`                     | Внутренний fan-out по §3.6 (по изменению статуса)                                                                      |
| `mail.scan.ready`                   | Скан письма готов к просмотру                                                                                                                                                                                                                                                    | `message_id`                              | Внутренний fan-out по §3.6 (`mail.scan_ready`)                                                                         |
| `delivery.ttn.manual`               | ТТН Новой почты внесена в 1С вручную; синхронизация кабинета и авто-запрос НП                                                                                                                                                                                                    | `message_id` + `ttn_number`               | Нет (обновление данных в кабинете)                                                                                     |
| `document.create`                   | Создать документ в кабинете; опц. `extension_request_id` + `extension_meta` — счёт по заявке продления (§3.9)                                                                                                                                                                   | `document_id`                             | Внутренний fan-out по §3.6 (`document.new`)                                                                            |
| `notification.push`                 | Универсальное уведомление из 1С (оплата, договор, бух. документы, лист заботы, просрочка, Укрпочта и т.д.); fan-out в каналы — на стороне сайта                                                                                                                                  | `notification_id` (fallback `dedupe_key`) | Fan-out по §3.6 (по `notification_code`)                                                                               |
| `notification.resolve`              | Снять активное уведомление (повод неактуален)                                                                                                                                                                                                                                    | `notification_id` / `dedupe_key`          | Снятие записи у пользователя                                                                                           |


### OUT: Site → 1С


| Действие                     | Метод | URL                                    | Триггер                                                                                                                                                                                                                                                                                                                                              | Идемпотентность                                          |
| ---------------------------- | ----- | -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| Проверка пользователя        | GET   | `/users/{login}/`                      | Регистрация, вход                                                                                                                                                                                                                                                                                                                                    | — (чтение)                                               |
| Синхронизация в 1С           | POST  | `/users/sync/`                         | Изменения из кабинета: контакты и атрибуты компании по `login` (`name`, `organization_name`, `scan_rule`). Поле `**changed_at`** обязательно — LWW против IN `user.update` (§3.1). Если передан `**scan_rule**`, дополнительно обязательно `**changed_by**` (§4.2). Первичная установка `scan_rule` с сайта запрещена до записи в 1С → ответ **409** | `login` (LWW по `changed_at` относительно `user.update`) |
| Синхронизация корп. юзера    | POST  | `/corporate/users/sync/`               | Регистрация и/или привязка к компании                                                                                                                                                                                                                                                                                                                | `email` (+ `cabinet_id` для linked)                      |
| Sync доступа сотрудника      | POST  | `/corporate/users/access/`             | Создание membership через сайт со стартовой видимостью / правка доступов мастером в UI (двусторонняя связь, IN-копия `corporate.user.access.update` — §3.2 / §4.9)                                                                                                                                                                                   | `cabinet_id + email` (snapshot, LWW по `synced_at`)      |
| Удаление сотрудника          | POST  | `/corporate/users/membership/removed/` | Мастер кабинета удалил сотрудника через UI сайта (см. §4.10); master не удаляется — `409 master_cannot_be_removed`                                                                                                                                                                                                                                   | `cabinet_id + email`                                     |
| Заказ скана письма (ТЗ §4.2) | POST  | `/mails/scan-request/`                 | Клиент нажал «Заказать скан»                                                                                                                                                                                                                                                                                                                         | `request_id` / `message_id`                              |
| Запрос акта сверки           | POST  | `/documents/reconciliation/`           | Клиент в кабинете (**async**, ответ `202`)                                                                                                                                                                                                                                                                                                           | `request_id`                                             |
| Заявка на продление договора | POST  | `/contracts/extension/`                | Запрос счёта в диалоге (асинхронно, ответ **202**); PDF счёта — через **`POST /intake/`**, **`document.create`** с **`extension_request_id`** = **`request_id`** (§3.9 / §4.8)                                                                                                                                                                         | `request_id`                                              |
| Список организаций           | GET   | `/organizations/{login}/`              | Открытие кабинета                                                                                                                                                                                                                                                                                                                                    | — (чтение)                                               |
| ТТН создано (после API НП)   | POST  | `/delivery/ttn/`                       | После успешного вызова API Новой почты; при заказе из **корп. кабинета** в теле `**cabinet_id`** + `**initiated_by_email**` (кто нажал «Заказать доставку»), см. §4.7.1                                                                                                                                                                              | `ttn_number` (+ `message_ids`)                           |
| Статус доставки (НП → сайт)  | POST  | `/delivery/status/`                    | Изменение `delivery_status` отправления в НП                                                                                                                                                                                                                                                                                                         | `ttn_number + delivery_status`                           |
